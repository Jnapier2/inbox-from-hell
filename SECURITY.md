# Security Policy

## Supported version

Security fixes apply to the current version on the default branch and the linked live demo.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability. Use GitHub's private vulnerability reporting for this repository when available, or use the contact channel on the official portfolio page.

Include the affected browser and version, a concise reproduction, the expected and observed behavior, and any potential impact. Do not include real credentials, personal data, or destructive proof-of-concept material.

Reports will be reviewed before details are published. This project has no server-side application, accounts, telemetry, or external API calls; the primary security boundary is the static browser runtime and its local save data.
